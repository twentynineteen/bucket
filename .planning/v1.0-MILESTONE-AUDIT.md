### Integration Issues (non-blocking tech debt)

1. **Dual copyPremiereProject** — BuildProject/api.ts and Premiere/api.ts both wrap the same `copy_premiere_project` Rust command independently. Premiere barrel only exports `PremierePluginManager`, so BuildProject cannot consume Premiere's wrapper through the barrel. If the Rust command signature changes, both wrappers must be updated.

2. **TrelloCard type collision** — Two different `TrelloCard` interfaces: Baker barrel re-exports the breadcrumbs shape (`url, cardId, title`) from `@shared/types`; Trello barrel exports the API response shape (`id, name, desc, idList`). Already mitigated with `LegacyTrelloCard` alias in `useAppendBreadcrumbs.ts`.

3. **Orphaned Trello barrel exports** — `TrelloIntegrationButton` and `TrelloIntegrationModal` are exported from `@features/Trello` barrel but have zero external consumers. Candidates for barrel removal.
